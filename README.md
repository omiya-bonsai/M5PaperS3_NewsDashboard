# M5PaperS3 News Viewer (PNG Remote Slide Client)

## 概要

本プロジェクトは **M5Stack M5PaperS3** を用いて、Raspberry Pi
等で生成された ニュース画像（PNG）を取得・表示する
**電子ペーパー型ニュースビューア**です。

HTTP 配信される複数ページの PNG を読み込み、

-   スワイプ
-   タップ
-   タイトルタップ
-   キャッシュ制御
-   version ファイルによる差分検出

によって高速・低消費電力で動作します。

------------------------------------------------------------------------

# 最新アーキテクチャ（重要）

現在の構成は次のように整理されています。

Raspberry Pi:

-   RSS取得
-   PNG生成
-   index.version生成
-   HTTP配信

M5PaperS3:

-   index.version確認
-   差分検出時のみ index.png 更新
-   詳細ページは SD キャッシュ利用
-   タップ操作でページジャンプ

この構造により通信量と描画回数を最小化しています。

# index.version 差分更新方式

最初に取得するのは画像ではありません。

まず

    index.version

のみ取得します。

内容例:

    20260326-1910

前回取得値と一致すれば

    index.png は再取得しない

つまり:

-   通信削減
-   消費電力削減
-   e-paper更新回数削減

を同時に実現します。

右スワイプ:

------------------------------------------------------------------------

# タップ操作

一覧画面:

    各ヘッドラインを直接タップ
    → 対応記事ページへジャンプ

詳細画面:

    タイトルタップ
    → index.png に戻る

右上タップ:

    強制更新（キャッシュ無視）

------------------------------------------------------------------------

# SDカードキャッシュ機構

以下のファイルを SD に保存します:

    /cache/index.png
    /cache/page1.png
    /cache/page2.png
    /cache/page3.png
    /cache/page4.png
    /cache/page5.png
    /cache/page6.png
    /cache/index.version

一定時間以内なら

    WiFiアクセスなし

でページ遷移します。

体感速度が大幅に改善されます。

------------------------------------------------------------------------

# 自動更新ポリシー

既定値:

    5分ごと

処理内容:

1 index.version取得 2 変更ありなら index.png 更新 3
変更なしなら何もしない

------------------------------------------------------------------------

# フッター表示内容

例:

    LAST 03-26 19:10   RF1 PF0   P2/7 BAT 78%

意味:

  表示   内容
  ------ --------------------
  LAST   最終成功更新時刻
  RF     更新失敗回数
  PF     ページ取得失敗回数
  P2/7   ページ番号
  BAT    バッテリー残量

20%未満:

    強調表示

# トラブルシューティング

## index.png 更新されない

index.version が変化しているか確認:

    curl http://server:8010/index.version

------------------------------------------------------------------------

## WiFi接続失敗

確認:

-   SSID
-   PASSWORD
-   2.4GHz利用可否

------------------------------------------------------------------------

## SD初期化失敗

確認:

-   FAT32形式
-   接触不良
-   別カード検証

------------------------------------------------------------------------

## LAST が更新されない

意味:

    HTTP取得失敗
    または
    server停止
