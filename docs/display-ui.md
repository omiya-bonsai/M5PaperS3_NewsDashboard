# Display UI

## 画面構成

M5PaperS3 の表示は、大きく次の 3 要素でできています。

- ベース画像
  - `index.png` または `page1.png` など
- 右上ラベル
  - `index NEW` / `page1 READ` など
- フッター
  - 状態、時刻、ページ、電池情報

## ベース画像

画像本体は Raspberry Pi 側で生成した PNG を表示します。

- `index.png`
  - 一覧画面
- `page1.png` から `page6.png`
  - 詳細画面

M5 側は画像の再生成はせず、SD 上の PNG を描画します。

## 右上ラベル

右上には、現在ページと既読状態をまとめたラベルを表示します。

例:

- `index NEW`
- `index READ`
- `page1 NEW`
- `page3 READ`

### 色ルール

- `NEW`
  - 黒地白文字
- `READ`
  - 白地黒文字

これは「未読を強く目立たせる」ためです。

## 既読 / 未読の意味

既読状態は記事単位ではなく、`index.version` 単位で管理します。

- 新しい `index.version` を受信
  - `NEW`
- タップ / スワイプ / 手動更新などの有効操作を行う
  - `READ`

つまり「今回の更新一式をもう触ったかどうか」を表します。

## フッター

最下部には 3 分割の情報帯があります。

### 左

- `LAST mm-dd hh:mm`

最後に正常更新できた時刻です。

### 中央

- `LIVE 30m`
- `CACHED 30m`
- `HTTP 30m`

のように、状態コードと更新間隔を表示します。

### 右

- `P0/6`
- `BAT xx%`
- `USB`

現在ページとバッテリー状態を表示します。  
USB 給電検出時は `USB` が付きます。

## 低電池時の表示

低電池時は、右側の電池情報領域を黒背景で強調します。

## 描画方針

- 画像表示後にフッターと右上ラベルを重ねる
- 通信中でも通常は全画面を白クリアしない
- 失敗時も既存画面を維持する

これにより e-paper 特有の「途中状態が残る」問題を減らしています。
